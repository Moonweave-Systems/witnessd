# W11 — Planner/Orchestrator: goal → sealed plan → dispatch (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans. TDD, `- [ ]` checkboxes.
> 스펙 근거: `SPEC.md` §2.4.1(인터페이스·검증 기준 원문) + `SPEC2.md` §3 (D11-1~D11-4).

**Goal:** v1.0의 마지막 스펙 갭 — 지금은 사람이 `--lane "L1:adapter=..."`로 팀을 손조립한다.
W11은 `plan(goal) -> list[LanePacket]`을 구현해 "목표를 던지면 팀이 짜인다"를 만든다.
**plan 자체가 증거다**: 채택된 plan은 canonical_hash로 봉인된 artifact가 되고, dispatch는 그
봉인본의 순수 함수다(D11-1). 완료 정의 = committed sealed-plan fixture를 Depone이 재도출 +
동일 봉인 plan → 동일 dispatch 이벤트 열(결정성).

**불변식:** Planner는 spawn·git·서명·verdict 금지(§2.4.1) / region disjoint 아니면 fail-closed
(merge-lane 명시 생성만 허용, D11-3) / LanePacket은 W7 레인 스키마의 상위집합(새 실행 경로 발명
금지 — `run_team`에 그대로 투입) / runtime depone import 금지.

## Task 1: LanePacket + seal (RED→GREEN)
- [ ] RED: `LanePacket` dict 스키마 `{lane_id, adapter, tier, region[], prompt, budget{max_tokens,
  max_usd,max_depth}, stop_rule}` 검증기 — W7 `_parse_team_lane` 산출과 왕복 변환 가능해야 함
  (packet→lane spec→packet 항등). adapter 없는 packet은 셸 레인으로 강등 아님 — 명시 `adapter:"shell"`.
- [ ] RED: `seal_plan(packets) -> {kind:"witnessd-sealed-plan", schema_version, goal, packets,
  plan_hash}` — plan_hash는 canonical_hash(packets). region overlap이 있는데 merge-lane이 없으면
  `PlannerError("ERR_PLAN_REGION_OVERLAP")` fail-closed.
- [ ] GREEN + 커밋.

## Task 2: 휴리스틱 planner (결정적 경로)
- [ ] RED: `plan_heuristic(goal, seed, root) -> list[LanePacket]` — LLM 없이 goal 문자열과 repo

- [ ] GREEN + 커밋.

## Task 4: dispatch + plan-run 배선
- [ ] RED: `dispatch(sealed_plan) -> list[DispatchEvent]` 순수 함수 — 동일 봉인본 2회 호출 →

- [ ] GREEN + 커밋.
