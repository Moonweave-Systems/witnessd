# v2.0.0 — 최종판 릴리스: one-command 실증 + 잔여 스윕 + 태그 (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.
> 스펙 근거: `SPEC2.md` §5(수용 기준 — 이 웨이브의 완료 정의). 이 웨이브는 **굳히기 + 실증**이다:
> 새 기능 없음. W10(진짜 에이전트)·W11(planner)·W12(진짜 A2)는 각각 증명됐으나, **"한 커맨드로
> 전 과정이 묶여 도는 것"은 아직 실증 안 됨** — 그게 이 웨이브의 헤드라인 게이트다.

**Goal:** SPEC2 §5 수용 기준을 실제로 충족하고 `v2.0.0`을 태그한다:
1. `witnessd team plan-run "<goal>"` 한 커맨드가 plan 봉인 → 레인 dispatch → **진짜 에이전트 실행**
   → 증거 트리 → evidence-pending 종료까지 도달.
2. 그 run의 증거 트리를 Depone이 오프라인 재도출(plan hash + 레인 A1/(격리 레인)A2 + team-ledger
   verdict + 서명 전부).
3. `fixtures/v2-demo/`에 그 증거 트리 committed + `revalidate_v2_demo.py` PASS.

**불변식:** runtime stdlib+openssl only / evidence-pending(자기 verdict 금지) / worker self-seal 금지 /
push·태그 push는 운영자 / production_gate 불변 / depone 불변 / 과대주장 금지.

## Task 1: one-command 실증 (수용 기준 1·2 — 유료 실행, 가드)
- [ ] **먼저 무비용 리허설**: fake codex 바이너리로 `team plan-run`이 plan→dispatch→run_team→
  evidence-pending까지 도는 것을 테스트로 고정(`tests/test_v2_plan_run.py`). 여기서 배선 버그를 다 잡는다.
- [ ] **진짜 실행 1회(가드)**: 작은 goal(예: "이 실패 테스트 통과시켜라")로
  `witnessd team plan-run "<goal>" --draft-adapter=heuristic --lane-adapter codex`(또는 plan이 codex
  레인을 생성) → 진짜 codex 레인 1개가 실제 코드 작성. **예산 플래그로 캡**(max-usd/tokens/depth 보수적).
  실패 또는 trivial diff면 프롬프트 고쳐 **최대 1회 재시도**, 2회 실패 시 STOP·보고.
  codex를 워커로 몰 때 W4 상태격리(별도 state-root) 강제 — 불가하면 claude 폴백.
- [ ] 출력이 evidence-pending인지(자기 verdict 아님), 레인 diff가 진짜 코드인지 확인.

## Task 2: 증거 트리 봉인 + Depone 전과정 재도출 (수용 기준 3)
- [ ] 그 run의 plan artifact + 레인 evidence + team-ledger를 `fixtures/v2-demo/`에 커밋(공개키만,
  개인키·크리덴셜 스캔 0). W10처럼 path-bound 한계가 있으면 정직 표기.
- [ ] `scripts/revalidate_v2_demo.py`: Depone이 committed 바이트에서 (a) plan_hash 재계산 일치,
  (b) 각 레인 capture-manifest/서명/runner-receipt 재도출, (c) team-ledger verdict 재도출을 모두 PASS.
  negative 1건 이상(예: 레인 서명 위조 → 거부).
- [ ] 전체 회귀 그린: unittest + revalidate w1~w12,v2_demo,key_rotation.
