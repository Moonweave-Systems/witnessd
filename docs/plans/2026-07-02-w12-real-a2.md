# W12 — Real A2: uid 격리를 실제로 세우고 W1 A2를 strict 증거로 전환 (Implementation Plan)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans.
> 스펙 근거: `SPEC.md` §3.4(uid 모델)·Part I Decision(uid 1급) + `SPEC2.md` §4 (D12-1~D12-3).
> **주의: 이 웨이브는 sudo가 필요한 호스트 작업 포함 — 해당 스텝은 운영자 승인/실행.**

**Goal:** A2(`A2-isolated-observed`)는 제품 assurance 사다리의 최상단이다. 이 웨이브는 전용
observer 계정을 실제로 만들어(호스트 sudo 가능 실측 확인됨, 2026-07-02) 진짜 격리 run이
A2를 방출하고 Depone strict assert가 통과하게 한 뒤, **그 증거로만** W1 A2를 strict fixture로
전환한다.

**불변식:** 문서 먼저 제거 금지(D12-2 — 증거가 먼저) / `isolation.py` 기존 fail-closed 로직 재사용
(재작성 금지) / observer_dir 0700·runner 쓰기 불가 / root(uid 0) 실행은 기존대로 fail-closed 유지.

## Task 0: 현재 격리 판정 로직 실측
- [ ] `witnessd/isolation.py`(`verify_isolation_boundary`, `probe_lane_isolation`)와 Depone
  `agent_fabric/isolation.py`의 A2 판정 조건을 실코드로 확정(추측 금지): 요구 facts 필드,
  `runner_uid != observer_uid != 0`, observer_dir 권한 검사.
- [x] 기존 W1 A2 fixture를 `fixtures/w12/capture-manifest.json`의 real A2 bytes와 byte-for-byte
  동일하게 전환한다.

## Task 1: 호스트 셋업 (운영자 스텝 — 에이전트는 스크립트만 준비)
- [ ] `scripts/setup_a2_host.sh` 작성(멱등): `witnessd-observer` 시스템 계정 생성,
  observer_dir 소유·0700, runner(ubuntu uid 1001)가 쓰기 불가 확인, sudoers 최소 항목
  (observer로 관측 프로세스 기동에 필요한 최소 권한만) 문서화. **스크립트는 에이전트가 작성,
  실행은 운영자가 검토 후**(`sudo bash scripts/setup_a2_host.sh`).
- [ ] 검증 스텝 포함: 셋업 후 `uv run python3 -m witnessd isolation --self-test`가 실격리 조건에서
  통과.
