# ORRO sketch method

Use this knowledge body before `flowplan` when the implementation direction is
not yet explicit. It is planning context only: not proof, verifier truth,
approval, or assurance, and it cannot change an evidence verdict.

## Method

1. Frame the problem in terms of the desired observable outcome, constraints,
   affected people or systems, and what success would make true.
2. Diverge into at least three genuinely distinct implementation shapes. Each
   candidate must state benefits, risks, and the tradeoff it accepts; renaming
   one design three times does not count.
3. Converge on one direction. Selection requires a reason tied to repository
   signals and constraints, not taste or model confidence.
4. Convert every remaining material choice into a decision branch with exactly
   one recommended answer and rationale. Mark it pending operator confirmation
   when the operator has not decided it; never leave an unranked menu.
5. Hand the chosen direction, constraints, and recommended branch resolutions to
   `orro flowplan` as intent. The sketch artifact remains advisory after handoff.

## Gate into the evidence pipeline

`sketch -> flowplan` is allowed when a chosen direction and flowplan input are
present. Execution begins only at `proofrun`; evidence truth begins only when
Depone evaluates persisted run artifacts through `proofcheck`.
